public class Operation extends Exception{
    public static double getresult(double numberA,double numberB,String operation)throws Exception
    {
        double result=0d;
        switch (operation){
            case "+":
                result=numberA+numberB;
                break;
            case "-":
                result=numberA-numberB;
                break;
            case "*":
                result=numberA*numberB;
                break;
            case "/":
                if(numberB==0){
                    throw new Exception("除数不可为0");
                }
                result=numberA/numberB;
                break;
        }
        return result;
    }

}

public class Operation1 {
    public double _numberA=0;
    public double _numberB=0;

    
    public double getResult()throws Exception{
        double result=0;
        return result;
    }

}

加减乘除子类

public class OperationAdd extends Operation1{

    @Override
    public double getResult(){
            double result=this._numberA+this._numberB;
            return result;
    }

}
public class OperationSub extends Operation1{
    @Override
    public double getResult() throws Exception{
        double result=this._numberA-this._numberB;
        return result;
    }
}
public class OperationMul extends Operation1{
    @Override
    public double getResult() throws Exception{
        double result=this._numberA*this._numberB;
        return result;
    }
}
public class OperationDiv extends Operation1{
    @Override
    public double getResult() throws Exception {
        if (this._numberB==0){
            throw new Exception("除数不能为0");
        }
        double result=this._numberA/this._numberB;
        return result;
    }
}

运算工厂类

public class OperationFacotry {
    public static Operation1 createOperation(String operation){
        Operation1 op=null;
        switch (operation){
            case "+":
                op=new OperationAdd();
                break;
            case "-":
                op=new OperationSub();
                break;
            case "*":
                op=new OperationMul();
                break;
            case "/":
                op=new OperationDiv();
                break;
        }
        return op;
    }
}

客户端

public class ma1 {
    public static void main(String[] args) {
        try{
          Operation1 oper;
          Scanner sc=new Scanner(System.in);
          System.out.println("请输入运算符号:");
          String strOpreation = sc.next();
          oper=OperationFacotry.createOperation(strOpreation);
          System.out.println("请输入数字A:");
          Double strNumberA= sc.nextDouble();
          oper._numberA=strNumberA;

            System.out.println("您的输入有误:"+e.getMessage());
        }
    }

}

工厂模式小结：

单例模式

某个类只能存在一个对象实例，并且该类只提供一个取得其对象实例的方法（静态方法）

源码实例: Hibernate的SessionFactory，充当数据存储源的代理，并负责创建Session对象，由于sessionFactory是一个重量级对象，通常情况下一个项目只需要一个，这就是单例模式。

饿汉式：

懒汉式（线程不安全）

在调用getInstance方法时才会创建单例对象，起到了懒加载效果，只不过线程不安全

懒汉式（线程安全，同步方法）

保证了线程安全，但是效率太低，每一次调用都要实例化对象

//提供一个静态的公有方法，加入同步处理的代码，解决线程安全问题
//即懒汉式
    public static synchronized Singleton getInstance() {
        if(instance == null) {
            instance = new Singleton();
        }
        return instance;
    }

双重检查

//提供一个静态的公有方法，加入双重检查代码，解决线程安全问题, 同时解决懒加载问题//同时保证了效率, 推荐使用    public static synchronized Singleton getInstance() {        if(instance == null) {            synchronized (Singleton.class) {                if(instance == null) {                    instance = new Singleton();                }            }        }        return instance;    }

这种方式实例化代码只执行一次，效率大大提高！

实现了懒加载，采用了类装载的机制来保证初始化实例时只有一个线程；在创建instance对象时，jvm底层机制保证了线程安全，在类进行初始化时，其他线程是无法进入的。

原型模式

在spring源码中的应用

在xml文件中配置bean的属性时，可以为prototype，可以为singleton，prototype就为原型模式，多次创建的是不同的对象。

浅拷贝

总结：如果为引用型变量就是复制地址，指向的就是同一个对象。

深拷贝

总结：为引用类型重新开辟存储空间

深拷贝推荐方式2，通过对象的序列化反序列化来实现，因为类中的引用类型变量有可能很多。

注意：

重复创建对象利用原型模式可以提高效率，可以动态获得对象的状态

缺点，在已有的类配备克隆方法时，违背了ocp原则

建造者模式

又叫做生成器模式。可以将复杂对象的建造过程抽象出来，使这个抽象工程的不同实现方法可以构造出不同表现的对象，

用户只指定复杂对象的类型和内容即可构建，不需知道内部的具体构建细节。

jdk源码应用：

注意：

1.客户不必知道产品内部细节，将产品本身与创建过程解耦，使得相同的创建工程可以创建不同的产品

2.用户使用不同的建造者可以的得到不同的产品对象。

3.增加新的具体建造者无需修改原有代码，符合ocp原则

接口适配器

适用于一些接口不想全部实现其中的方法

因为实现类接口必须实现所有方法，但是继承一个抽象类不需全部实现

所以就选一个抽象类实现接口，然后再继承抽象类，想实现哪个方法，进行覆盖就可以

这个抽象类就充当适配器角色。
